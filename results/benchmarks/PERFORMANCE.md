## Notes

- ``in_memory`` is the default ``InMemoryStorage`` backend (state lost on process exit; no persistence overhead).
- ``sqlite_memory`` is ``SQLiteStorage(":memory:")`` -- exercises the SQL serialization path without disk I/O.
- ``sqlite_disk`` is ``SQLiteStorage(tempfile)`` -- includes disk I/O and represents a typical production deployment.
- ``hash`` embeddings are the deterministic 64-dim hash-bag-of-words used in the validation harness. ``sentence_transformers`` uses ``all-MiniLM-L6-v2`` (384-dim).
- ``query`` operations time the full pipeline: retrieve -> fold -> route. ``add_derived`` includes both contributing-input resolution and provenance propagation. ``audit`` walks the current derived-record set.
